Understanding the Workflow

This page documents a real coding session where Claude Code built a complete macOS application. The key to this workflow is a powerful command combination:

$ yoloclaude # aliased to: claude --dangerously-skip-permissions

--dangerously-skip-permissions enables fully autonomous execution after plan approval.

Then press Shift+Tab three times to enter Plan Mode before sending the prompt.

Human

What Humans Do

  • • Start Claude with the right flags
  • • Write detailed requirements
  • • Review and approve the plan
  • • Watch it build (optional)
  • • Test the final result
Autonomous

What Claude Does

  • • Explore existing codebase
  • • Design architecture
  • • Write implementation plan
  • • Create all source files
  • • Build, debug, and fix errors

Key Takeaways

  • ★ --dangerously-skip-permissions is what makes this workflow possible - approve once, implement fully autonomously
  • ✓ Plan Mode (Shift+Tab) lets Claude research and design before coding - you approve the approach first
  • ✓ Sub-agents handle specialized tasks (exploration, architecture) autonomously
  • ✓ Todo tracking gives visibility into what Claude is doing at any moment
  • ✓ Self-debugging means Claude fixes its own compilation errors without help
Human Action

The Initial Prompt

The human started Claude Code in Plan Mode and entered this detailed prompt. Notice how it specifies the technical requirements clearly but leaves implementation details to Claude:

H
Human 14:47:34 UTC

Create a macOS app written in Swift, using SwiftUI and AVFoundation, that synthesizes rain sounds.

The audio engine should manage a number of raindrops. The raindrops should be created with some minor randomization of timing but the average rate should be controllable using a slider in the user-interface so that average rates between 20 per minute and 4000 per minute are possible.

Each raindrop should be synthesised using AVAudioSourceNode and the generation should involve a separate "impact" sound, followed by a small delay, followed by an exponentially decaying "bubble" oscillation. The user interface should include sliders to adjust the impact amplitude, the delay duration, the bubble amplitude, the bubble frequency and the bubble decay rate. As with the timing, each of these parameters should have minor randomisation so the drops do not all sound mechanically identical. As the sliders are adjusted, a SwiftUI Chart should show a sample waveform for the raindrop based on the current settings.

To fill out the overall "rain" soundscape, the audio engine should also implement pink, brown and white noise generators whose amplitude can also be controlled via sliders in the user interface.

I used Xcode to create the template project here, so it should build and run properly. But it's just the basic from Xcode, so use that as the basis. Don't bother writing any tests though. Right now we want to focus just on the functionality.

The Raindrop Synthesis Formula
raindrop(t) = impact(t) + bubble(t - delay)

impact(t) = A_impact * e^(-t/tau_impact) * noise(t)
bubble(t) = A_bubble * e^(-t/tau_bubble) * sin(2*pi*f_bubble*t)

Sample Code: Raindrop Synthesis

func generateSample() -> Float {
    var output: Float = 0.0

    // Impact phase - exponentially decaying noise
    let impactDurationSamples: UInt32 = 50
    if sampleIndex < delaySamples + impactDurationSamples {
        noiseState ^= noiseState << 13
        noiseState ^= noiseState >> 17
        noiseState ^= noiseState << 5
        let noise = Float(Int32(bitPattern: noiseState)) / Float(Int32.max)
        output += impactAmplitude * currentImpactEnvelope * noise
        currentImpactEnvelope *= impactDecayRate
    }

    // Bubble phase - exponentially decaying sine wave
    if sampleIndex >= delaySamples {
        output += bubbleAmplitude * currentBubbleEnvelope * sin(bubblePhase)
        bubblePhase += bubblePhaseIncrement
        if bubblePhase > 2.0 * .pi {
            bubblePhase -= 2.0 * .pi
        }
        currentBubbleEnvelope *= bubbleDecayRate
    }

    sampleIndex += 1
    return output
}

Sigh - Rain Sound Synthesizer

A fully functional macOS application that generates realistic rain sounds through procedural audio synthesis. Features include:

  • ● Raindrop rate: 20-4000 drops/minute
  • ● Impact + delay + bubble synthesis
  • ● Parameter randomization (+/- 15%)
  • ● Real-time waveform preview
  • ● Pink noise (Voss-McCartney)
  • ● Brown noise (integrated white)
  • ● White noise (XORShift)
  • ● Play/pause control

Tips for Using Claude Code

Do This

  • ✓ Use --dangerously-skip-permissions + Plan Mode
  • ✓ Give detailed requirements upfront
  • ✓ Review plans carefully before approving
  • ✓ Watch the todo list for progress
  • ✓ Let Claude debug its own errors

Avoid This

  • ✗ Interrupting mid-implementation
  • ✗ Vague or incomplete requirements
  • ✗ Skipping the planning phase
  • ✗ Not providing context about existing code
  • ✗ Expecting perfection on first attempt
